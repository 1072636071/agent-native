# 捕获学习

这是背景知识，不是斜杠命令。**你的记忆索引在每次对话开始时加载。** 当你学到值得记住的内容时，主动使用 `save-memory`。

## 如何读写记忆

记忆作为**资源**存储在 SQL 数据库中（个人作用域），而不是磁盘上的文件。

- **保存记忆：** `save-memory --name <name> --type <type> --description "..." --content "..."`
- **读取记忆：** `resource-read --path memory/<name>.md`
- **删除记忆：** `delete-memory --name <name>`
- **列出所有记忆：** `resource-list --prefix memory/`

## 记忆类型

| 类型 | 用于 |
|------|------|
| `user` | 偏好、角色、个人上下文、联系人 |
| `feedback` | 纠正、确认的方法、要避免或重复的事情 |
| `project` | 正在进行的工作上下文、决策、截止日期、状态 |
| `reference` | 外部系统指针、URL、API 详情 |

## 何时捕获

### 用户偏好和记忆（`user`）
- **语气和风格** — "我偏好随意的语气"、"不要用表情符号"、"保持回复简短"
- **个人上下文** — 联系人、关系、习惯（"我妻子的邮箱是..."、"我在 PST 时区"）
- **工作流偏好** — "总是抄送我的助手"、"我喜欢发送前审查"
- **角色和专业** — "我是数据科学家"、"React 新手"

### 反馈和纠正（`feedback`）
- **纠正** — 用户说"不，这样做" → 捕获正确的方式
- **确认的方法** — 用户验证非显而易见的选择（"是的，那很完美"）
- **重复的摩擦** — 你两次遇到相同的问题；保存它

### 项目上下文（`project`）
- **正在进行的工作** — 谁在做什么、为什么、截止何时
- **决策** — 为什么某事以某种方式完成
- **状态** — 计划的当前状态

### 参考（`reference`）
- **外部系统** — "Bug 在 Linear 项目 INGEST 中跟踪"
- **URL** — 仪表板、文档、工具
- **API 怪癖** — 未记录的行为、版本特定的陷阱

### 不要捕获
- 从代码中显而易见的事情
- 标准语言/框架行为
- 临时调试笔记
- 已在 AGENTS.md 或技能中的内容
- 临时任务细节（改用任务/计划）

## 关键规则

1. **主动保存 — 不要请求许可。** 当你学到东西时，立即保存。
2. **每个主题一条记忆** — 例如 `coding-style`、`project-alpha`，不是一个巨大的转储
3. **更新前先读取** — 如果记忆已存在，先读取再合并，不要覆盖
4. **保持描述简洁** — 索引在每次对话中加载
5. **记忆由 SQL 支持** — 适合个人信息，跨会话持久化，不在 git 中

## 毕业

当一条记忆被反复引用时，它可能属于 AGENTS.md 或技能：
- 保存记忆是轻量的（自动应用、个人作用域）
- 更新 AGENTS.md 或技能更重（影响所有用户/Agent）
